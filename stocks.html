02 / How It Works

From API call to published dashboard.

The project is a four-stage pipeline. Each stage hands a clean artifact to the next — the same pattern used for a production data feed, scaled down to a single ticker.

01 · FETCH

Pull from the API

A Python script calls the Alpha Vantage TIME_SERIES_DAILY endpoint for NVDA over HTTPS using requests, returning the latest 100 trading days as JSON.

02 · SHAPE

Clean & structure

pandas parses the JSON into a tidy table — Date, Open, High, Low, Close, Volume — typed, sorted ascending by date, and written to stock_data.csv.

03 · MODEL

Build in Tableau

Tableau reads the CSV. Level-of-detail calcs anchor the KPI tiles to the newest date, so current price and day-over-day change stay correct as the data refreshes.

04 · PUBLISH

Ship to the web

The data is extracted into the workbook and published to Tableau Public; this page embeds that live viz via the Tableau Embedding API.

03 / What's on the Dashboard

Five views of the same daily series.

Every tile is derived from the one OHLCV dataset. The figures reflect the published snapshot's last trading day; they update whenever the workbook is refreshed and re-published.

Current Price

The closing price on the most recent trading day in the data, resolved with a FIXED LOD on the max date.

Day Change %

Percentage move versus the prior trading day's close — signed, so declines read correctly in red.

Period High / Low

Highest high and lowest low across the ~5-month window. Labelled "period," not 52-week — the free daily feed only returns 100 sessions.

Price Trend

A line of daily closing prices across the full window, showing the recent trajectory.

Volume

Daily share volume as bars beneath the price line, for spotting conviction behind moves.
